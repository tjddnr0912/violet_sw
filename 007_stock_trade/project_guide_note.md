# Project Abstract
- 이 문서는 이번 프로젝트가 어떻게 구성되었으면 하는지에 대해서 기술한다.
- 이 프로젝트는 한국의 증권사중 1군데를 기반으로 API key를 발급받고, 해당 API를 이용해서 local에서 동작 할 수 있는 자동 매매 프로그램을 만드는 것을 목표로한다.
- 이 프로젝트의 손실은 전적으로 사용자에게 있기 때문에 손해에 대한 법적 책임에서 자유롭게 코딩을 진행한다.

## 증권사 선정
- 한국의 증권사들 중에서 API key를 발급하고, 그것을 이용해서 원하는 정보의 export, 증권에 대해서 매수/매도를 할 수 있는 곳을 찾는다.
- 프로그램이 동작하는 OS는 macOS이기 때문에 반드시 macOS를 지원하는 증권사여야 한다.

## API key 테스트
- 발급받은 API는 전체 full test에는 시간이 오래 걸리기 때문에 모듈화 되어서 기능을 테스트 할 수 있어야 한다. 테스트 코드는 사전에 미리 작성되어야 하고, 유저가 돌리기 보단 AI를 이용해 자동 실행을 하는 형태로 진행이 될 예정이다.

## API를 이용한 정보 추출
- 증권사의 API를 이용해서 받을 수 있는 정보의 자료는 raw의 형태여야 하고, AI는 코딩 단계에 있어서 선정된 증권사의 API가 어떤 것들을 제공하는지 사용자에게 고지하고, 어떤 데이터를 사용할지 반드시 지정되어야한다.

## 매수/매도 시스템
- 기본적으로 매수/매도가 정상적으로 진행되는지에 대한 모듈의 구현 그리고 테스트가 진행되어야한다.

## 텔레그램 챗봇을 이용한 정보 전달 및 리포팅
- 프로그램은 텔레그램 챗봇을 이용해서 사용자에게 매수/매도/정정/취소 등에 대한 방법들에 대해서 각각 메세지를 제공해 사용자에게 보내야한다.
- 사용자는 프로그램을 통해서 상태를 체크 할 수 있고, 유사시에는 프로그램 내부에 대한 수정 권한 또한 가지고 있다.
