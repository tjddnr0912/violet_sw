# 퀀트 자본 미집행(현금 누수) 복구 설계 — A안 최소 배관 복구

- **작성일**: 2026-06-04
- **상태**: 설계 승인 대기 → 구현 예정
- **범위**: 007_stock_trade 주문 집행 경로 (전략/팩터 모델 불변)
- **모드**: VIRTUAL(모의투자) — 실손실 0, 자유롭게 수정 가능

## 1. 배경 / 진단

모의투자 계좌가 2026-02-10 ~ 06-04 약 4개월간 정체(−10.4%). 원인을 데이터로 추적한 결과 **팩터 모델 문제가 아니라 자본이 시장에 들어가지 못하는 "배관" 문제**로 확정.

증거(`data/quant/daily_history.json`, `transaction_journal.json`, `logs/daemon_*.log`):

| 지표 | 값 | 해석 |
|------|-----|------|
| 자산 | 10,609,298 → 9,507,322 (−10.4%) | 4개월 정체 |
| 실현손익 | −52,318원 (승률 46%, 평익 +27,848 / 평손 −24,691) | 매매 자체는 거의 본전 |
| 평균 현금비중 | 77% (중앙값 88%) | 시장에 거의 미진입 |
| 평균 보유종목 | 4.7개 / 목표 15개 | 3월엔 0종목·100% 현금 |

핵심 서사: **2월에 +7.8% 벌었다가 3월에 전량 정리 → 그 뒤 재투자 안 됨.** 88% 현금이면 어떤 좋은 전략도 0%를 낸다.

### 1.1 근본 원인 (코드로 확인)

1. **매수 루프가 막힌 슬롯을 현금으로 버림** (`order_executor.generate_rebalance_orders`, L119–158)
   - 매수 후보 = `상위15 − 보유`만 순회. 가드(쿨다운/섹터한도/점수차)에 막히면 `continue`로 **그 슬롯을 포기**하고 16·17위 적격 종목으로 **내려가지 않음**.
   - value·quality 팩터가 한국 금융·보험주를 상위로 올리는데, 섹터한도(3개)·쿨다운(20일)이 이들을 줄줄이 차단 → 슬롯이 현금으로 샌다.
   - 로그 확인(2026-06-01): 신한지주 쿨다운 스킵, 태광산업·한화생명·DL이앤씨 섹터한도 스킵 → "15개 목표 중 6개 보유, 기타 7건".

2. **리밸런싱이 월 1회뿐** (`schedule_handler.setup_schedule`)
   - 일일 탑업 훅 없음. 월중 손절로 생긴 현금이 다음 달까지 방치.

3. **(부수 발견, 범위 외) 레짐 감지 미배선**
   - `MarketAnalyzer.get_market_condition()`이 엔진 어디서도 호출되지 않음. Bull/Bear 동적조정·벤치마크 비교(`benchmark_return_pct`가 계속 0) 모두 죽은 코드. 즉 봇은 *방어적으로 현금을 든 게 아니라* 배관이 새서 현금이 쌓인 것. → B안에서 처리.

## 2. 목표 / 비목표

**목표**
- 평균 현금비중을 77% → ~5–15%로 낮춰 목표 15종목을 실제로 채운다.
- 가드(쿨다운·섹터한도·점수차)의 **보호 의도를 보존**하면서 슬롯 누수만 제거.
- 추가 매매비용(증권거래세) 없이 달성.

**비목표 (이번 변경에서 건드리지 않음)**
- 팩터 가중치/스크리닝 로직 (4팩터 Percentile Ranking 불변).
- 레짐 배선·벤치마크 비교 (B안).
- 손절/익절 파라미터.
- 리컨실러 전면 재설계 (C안).

## 3. 설계

### 변경 1 — 폴스루 슬롯필링
**파일**: `src/quant_modules/order_executor.py` · `generate_rebalance_orders` (L119–158 매수 후보 선정부)

- **현재**: `for code in (target_holdings - current_holdings)` → 가드 막히면 `continue`로 슬롯 포기.
- **변경**: 이미 계산돼 있는 `ranked`(= `screening_result.all_scores` 중 `passed_filter`, 점수 내림차순; L111–114)를 **상위부터 순회**하며 다음을 만족할 때까지 슬롯을 채운다:
  - 건너뛸 조건: 이미 보유 중 / 쿨다운 차단 / 섹터한도 초과.
  - 종료 조건: `len(유지+매수) == target_stock_count` 또는 `ranked` 소진.
  - 막힌 종목은 건너뛰되 **슬롯은 다음 적격 종목이 채운다**(폴스루).
- **점수차(P10, `min_score_gap=5.0`) 게이트**는 *기존 보유와의 교체* 판단에만 적용. *빈 슬롯 채우기*엔 미적용(빈 슬롯은 점수차로 막지 않음).
- **폴백**: `all_scores`가 비는 날(데이터 장애)엔 기존 `상위15 − 보유` 동작 유지.

검증 전제(확인됨): `screener.py:440`에서 `all_scores=scores`로 라이브 메모리에 항상 채워짐(저장 JSON만 잘림). 폴스루가 걸을 리스트 존재.

### 변경 2 — 매수전용 일일 탑업
**파일**: `src/quant_modules/schedule_handler.py` · `on_market_open` (L281) + 엔진 메서드 1개 신설(`quant_engine.py`)

- **현재**: `on_market_open`은 `_is_rebalance_day()`일 때만 매수.
- **변경**: 비리밸런싱 영업일에 다음 조건이면 **매수전용 탑업** 분기 추가:
  - 조건: `len(positions) < target_stock_count` **AND** 가용현금 ≥ 1종목치(`total_capital × min_weight(3%)`).
  - 동작: `engine.last_screening_result` 재사용(없거나 `last_screening_date`가 5영업일 초과 경과 시에만 `run_screening()` 재실행) → **변경 1의 폴스루 슬롯필링을 매수 주문만 생성**해 집행.
  - **매도/재정렬 없음.** 기존 보유는 건드리지 않고 빈 슬롯만 채운다.

### 3.1 가드 보존 근거
폴스루가 생기면 쿨다운·섹터한도가 더 이상 "슬롯을 낭비"하지 않는다(막힌 종목은 슬롯을 다음 적격 종목에 넘김). 따라서 **세 가드를 그대로 둔다**:
- 쿨다운 20일 (`COOLDOWN_DAYS`, L31) — 반복 손절 루프 방지 의도 보존.
- 섹터한도 3 (`DEFAULT_MAX_PER_SECTOR`, L35) — 2026-03-09 금융주 4종 동시손절 방지 의도 보존.
- 점수차 5 (`min_score_gap`, L121) — 미미한 교체 churn 방지.

(선택) 스크리너 `max_per_sector=0.30`(screener.py:34, ≈4–5/15)과 집행기 `3` 불일치는 폴스루로 무해해짐(스크리너가 4개 뽑아도 집행기가 3에서 막으면 슬롯은 다음 종목이 채움). 이번엔 정합화 불필요.

### 3.2 비용 고려
일일 탑업은 **매수 전용**(빈 슬롯만 채움), 매도 재정렬은 월 1회 리밸런싱에만. → 증권거래세(매도 0.18%) 미발생. 014에서 확인한 *비용-빈도 법칙* 준수(고빈도 매매 = 비용 사망).

## 5. 엣지 케이스
- `ranked` 적격 종목이 15개 미만(얇은 날): 채울 수 있는 만큼만 채우고 기존 `_check_position_shortage` 경고 유지.
- 가용현금이 1종목치 미만: 탑업 no-op.
- 모든 `ranked`가 이미 보유: no-op.
- 리밸런싱 당일: 기존 월간 경로만 실행(탑업 분기 미발화, 중복 방지).

## 6. 검증 계획
- **단위테스트 2종**
  1. 보유 + 쿨다운 + 섹터막힘이 섞인 `ranked`를 주면 → 정확히 `target_count`칸을 적격 종목으로 채우고, 막힌 종목은 건너뛰며 다음 종목이 슬롯을 메우는지.
  2. 일일 탑업이 매수 주문만 생성하고(매도 0), 가용현금 한도를 넘지 않으며, 보유<target일 때만 발화하는지.
- **VIRTUAL 1일 드라이런**: 평균 현금 77%→~10%대, 보유종목→~15 확인.
- **기대 효과**: 88% 현금 → ~5–15%. 팩터 모델이 비로소 표현됨. 출혈의 1차 원인 제거.
