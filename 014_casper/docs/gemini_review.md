# Gemini Code Review Report (Casper Trading Bot)

> **작성일:** 2026-04-04
> **대상:** Casper Trading Bot v1.0 (014_casper)
> **우선순위:** 1. 안전성 및 에러 대응, 2. 보안 및 논리적 무결성

---

## 1. 프로젝트 개요 및 의도
Casper Bot은 TQQQ/SQQQ를 대상으로 하는 인트라데이 자동매매 시스템입니다. ORB(Opening Range Breakout)와 FVG(Fair Value Gap) 전략을 결합하여 정교한 진입 시점을 포착하며, 엄격한 리스크 관리(VIX 필터, 주간 손실 제한, 서킷 브레이커)를 통해 자산을 보호하도록 설계되었습니다.

---

## 2. 종합 리뷰 결과 (요약)

### [안전성 및 에러 대응] - **CRITICAL**
- **매도 무한 시도 위험:** `bot.py`의 `_close_and_record` 로직에서 매도 실패 시 다음 틱에서 계속 시도함. API 장애 시 무한 루프 가능성.
- **체결 데이터 불일치:** 실전 매매 시 체결가 조회가 늦어지면(6초 초과) 계획가로 정산되어 실제 자산과 오차가 발생함.
- **상태 전이 경직성:** 데이터 부족 시 즉시 `DONE_TODAY`로 전이하여 당일 기회를 모두 상실함.

### [보안 및 논리 무결성] - **HIGH**
- **수수료 모델 단순화:** KIS의 최소 수수료 규정을 고려하지 않아 소액 매매 시 수익률 계산 오차 발생 가능.
- **시간대 처리:** 서머타임 변경에 대한 동적 대응 확인이 필요하며, 장 마감 강제 청산 시점의 데이터 유실 대응이 미흡함.

---

## 3. 모듈별 심층 분석 결과

### 3.1 Core Strategy (`orb.py`, `fvg.py`, `strategy.py`)
- **ORB 계산:** 15분 범위 산출을 위해 5분봉 3개가 필수이나, 하나라도 누락 시 당일 매매가 중단됨. 누락된 데이터를 대체하거나 다시 불러오는 로직 부재.
- **FVG 감지:** `Close > ORB High`를 돌파 기준으로 사용하나, 몸통(Body) 돌파 여부를 명시적으로 체크하지 않아 꼬리 돌파에 의한 가짜 신호(Fake-out)에 취약할 수 있음.
- **진입 로직:** `fvg.mid`를 진입 타겟으로 잡는 '되돌림(Pullback)' 방식은 정교하나, 강한 추세장에서 진입 기회를 놓칠 가능성이 높음.

### 3.2 Bot Control (`bot.py`)
- **크래시 복구:** 포지션 복구 시 신호 데이터(`TradeSignal`)를 가짜(Stub)로 채움. 이는 복구 후의 리스크 계산 및 사후 분석 데이터의 신뢰도를 떨어뜨림.
- **이벤트 루프:** `_tick` 마다 상태를 체크하지만, 예외 발생 시 `time.sleep(30)` 후 재시도함. 이때 중요한 청산 시점을 놓칠 위험이 있음.

### 3.3 Risk Management (`risk.py`)
- **서킷 브레이커:** 과거 매매 기록에서 CB 상태를 복구하는 로직은 우수하나, 기록 데이터 파싱 실패 시의 예외 처리가 부족함.
- **VIX 필터:** VIX 데이터 수집 실패 시 무한 대기 루프에 빠질 수 있으며, 대체 데이터 소스(yfinance 등)로의 전환 로직이 보이지 않음.

### 3.4 API Interaction (`kis_client.py`, `kis_order.py`)
- **주문 안전성:** `retry=False` 설정을 통해 주문 중복 발생을 원천 차단한 설계는 매우 우수함.
- **폴링 메커니즘:** 체결가 조회 시 3회 폴링은 실전에서 부족할 수 있음. 특히 장 초반 거래량 폭주 시 KIS 서버 지연에 대응하기 어려움.

## 4. 최종 권장 개선 사항

1. **매도 프로세스 강화:** 최대 매도 시도 횟수를 제한하고, 최종 실패 시 긴급 텔레그램 알림 후 프로세스 중단.
2. **동적 데이터 복구:** 프로세스 시작 시 스텁 데이터를 사용하는 대신, 당일 과거 캔들을 다시 조회하여 상태를 완벽히 재구축(Rebuild)하도록 개선.
3. **체결가 조회 최적화:** 체결가 조회를 비동기로 처리하거나 폴링 횟수/시간을 늘려 실제 수익률 계산의 정확도 향상.
4. **Panic Sell 기능:** 보유 포지션이 있을 때 강제 종료 시 시장가로 즉시 청산하는 명령줄 옵션(`--panic`) 추가.
