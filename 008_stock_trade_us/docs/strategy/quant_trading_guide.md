## 1. 퀀트 매매란?

### 정의

**퀀트(Quant)**는 'Quantitative(계량적인)'의 줄임말로, **수학, 통계, 프로그래밍을 활용해 객관적인 데이터에 기반하여 투자 결정을 내리는 방식**입니다.

```
전통적 투자: 감(感) + 뉴스 + 루머 → 주관적 판단
퀀트 투자:   데이터 + 알고리즘 + 규칙 → 객관적 판단
```

### 핵심 구성 요소

| 요소 | 설명 | 예시 |
|------|------|------|
| **데이터** | 투자 판단의 원재료 | 주가, 거래량, 재무제표, 금리 |
| **알고리즘** | 데이터를 분석하는 규칙 | RSI > 70이면 매도 |
| **포트폴리오** | 종목과 비중의 조합 | 20종목, 각 5% 비중 |

### 퀀트 투자의 장점

1. **감정 배제**: 공포와 탐욕에 흔들리지 않음
2. **일관성**: 동일한 규칙으로 반복 실행 가능
3. **검증 가능**: 백테스팅으로 과거 성과 확인
4. **자동화**: 프로그래밍으로 24시간 운영 가능

### 퀀트 투자의 한계

1. **과최적화 위험**: 과거에만 맞는 전략이 될 수 있음
2. **블랙스완**: 예측 불가능한 사건에 취약
3. **학습 곡선**: 프로그래밍/통계 지식 필요

> **TIP**: 퀀트 투자는 "돈을 버는 마법"이 아니라 "실수를 줄이는 체계"입니다.

## 2. 수익 구조 이해하기

### 수익의 원천

퀀트 매매에서 수익은 크게 세 가지 원천에서 발생합니다:

```
┌─────────────────────────────────────────────────────────┐
│                    수익의 원천                            │
├─────────────────────────────────────────────────────────┤
│  1. 시장 수익 (Beta)     : 시장 전체 상승에 따른 수익      │
│  2. 초과 수익 (Alpha)    : 전략으로 얻는 추가 수익         │
│  3. 차익 수익 (Arbitrage): 가격 불균형에서 얻는 수익       │
├─────────────────────────────────────────────────────────┤
│  총 수익 = Beta × 시장수익률 + Alpha + Arbitrage         │
└─────────────────────────────────────────────────────────┘
```

### 기대 수익률 계산

**기대 수익률 공식:**
```
기대 수익률 = (승률 × 평균 수익) - (패률 × 평균 손실)
```

**예시:**
- 승률: 60%, 평균 수익: 5%
- 패률: 40%, 평균 손실: 3%
- 기대 수익률 = (0.6 × 5%) - (0.4 × 3%) = 3% - 1.2% = **1.8%**

### 복리의 마법

| 연 수익률 | 10년 후 (복리) | 20년 후 (복리) |
|-----------|----------------|----------------|
| 5% | 1.63배 | 2.65배 |
| 10% | 2.59배 | 6.73배 |
| 15% | 4.05배 | 16.37배 |
| 20% | 6.19배 | 38.34배 |

> **핵심**: 퀀트 투자의 목표는 "대박"이 아니라 **"꾸준한 복리 수익"**입니다.

## 3. 퀀트 전략의 종류

### 3.1 가치 투자 (Value)

**개념**: 기업의 내재 가치보다 싸게 거래되는 종목에 투자

**핵심 지표:**
| 지표 | 계산식 | 의미 | 저평가 기준 |
|------|--------|------|-------------|
| PER | 주가 / 주당순이익 | 이익 대비 주가 | 10 이하 |
| PBR | 주가 / 주당순자산 | 자산 대비 주가 | 1 이하 |
| PSR | 주가 / 주당매출 | 매출 대비 주가 | 1 이하 |
| PCR | 주가 / 주당현금흐름 | 현금 대비 주가 | 5 이하 |

**API 활용:**
```
필요 데이터: 재무비율 API (FHKST66430300)
- PER, PBR, ROE 등 재무 지표 조회 가능
- 종목 스크리닝에 필수
```

### 3.2 모멘텀 (Momentum)

**개념**: 최근 상승세인 종목은 계속 상승하는 경향을 활용

**유형:**
| 유형 | 기간 | 특징 |
|------|------|------|
| 단기 모멘텀 | 1주 ~ 1개월 | 노이즈가 많음, 거래비용 높음 |
| 중기 모멘텀 | 3개월 ~ 12개월 | **가장 효과적 (학술적 검증)** |
| 장기 모멘텀 | 3년 ~ 5년 | 평균회귀로 전환될 수 있음 |

**계산 방법:**
```
12개월 모멘텀 = (현재가 - 12개월전 가격) / 12개월전 가격 × 100%

예시: 현재가 55,000원, 12개월전 50,000원
모멘텀 = (55,000 - 50,000) / 50,000 × 100% = 10%
```

**API 활용:**
```
필요 데이터: 기간별시세 API (FHKST03010100)
- 일/주/월/년봉 데이터로 과거 가격 조회
- 모멘텀 계산의 기본 데이터
```

### 3.3 평균 회귀 (Mean Reversion)

**개념**: 과도하게 하락한 종목은 원래 수준으로 회복하는 경향을 활용

**핵심 원리:**
```
주가가 이동평균에서 크게 이탈 → 다시 평균으로 회귀
```

**지표:**
- 볼린저밴드 이탈
- RSI 과매수/과매도
- 52주 신저가 종목

> **주의**: 평균 회귀는 "싸니까 산다"가 아니라 "회복 신호가 보일 때 산다"입니다.

### 3.4 퀄리티 (Quality)

**개념**: 재무 건전성이 우수한 기업에 투자

**핵심 지표:**
| 지표 | 계산식 | 의미 | 우량 기준 |
|------|--------|------|-----------|
| ROE | 순이익 / 자기자본 | 자본 효율성 | 15% 이상 |
| ROA | 순이익 / 총자산 | 자산 효율성 | 5% 이상 |
| 부채비율 | 부채 / 자기자본 | 재무 안정성 | 100% 이하 |
| 영업이익률 | 영업이익 / 매출 | 본업 수익성 | 10% 이상 |

### 3.5 멀티팩터 (Multi-Factor)

**개념**: 여러 팩터를 조합하여 더 안정적인 수익 추구

**조합 예시:**
```
통합 점수 = (가치 점수 × 0.3) + (모멘텀 점수 × 0.3) + (퀄리티 점수 × 0.4)
```

**장점:**
- 단일 팩터의 약점 보완
- 상관관계가 낮은 팩터 조합 시 분산 효과

## 4. 기술적 지표와 계산식

### 4.1 이동평균선 (MA)

**단순이동평균 (SMA):**
```
SMA(n) = (P1 + P2 + ... + Pn) / n

예시: 5일 SMA = (10,000 + 10,200 + 10,100 + 10,300 + 10,400) / 5 = 10,200원
```

**지수이동평균 (EMA):**
```
EMA(today) = Price(today) × k + EMA(yesterday) × (1-k)
k = 2 / (n + 1)

예시: 12일 EMA, k = 2/13 ≈ 0.154
```

**매매 신호:**
| 신호 | 조건 | 해석 |
|------|------|------|
| 골든크로스 | 단기MA > 장기MA (상향돌파) | 매수 신호 |
| 데드크로스 | 단기MA < 장기MA (하향돌파) | 매도 신호 |

**API 활용:**
```
필요 데이터: 일별 시세 API (FHKST01010400)
→ 종가 데이터로 직접 계산
→ 분봉 API (FHKST03010200)로 단기 신호 포착
```

### 4.2 RSI (상대강도지수)

**계산식:**
```
RS = 평균 상승폭 / 평균 하락폭 (보통 14일 기준)
RSI = 100 - (100 / (1 + RS))
```

**예시 계산:**
```
14일 중 상승일: 8일, 평균 상승폭 2%
14일 중 하락일: 6일, 평균 하락폭 1.5%
RS = 2 / 1.5 = 1.33
RSI = 100 - (100 / 2.33) = 57.1
```

**매매 기준:**

| RSI 수준 | 상태 | 전략 |
|----------|------|------|
| 70 이상 | 과매수 | 매도 검토 |
| 50 | 중립 | 관망 |
| 30 이하 | 과매도 | 매수 검토 |

> **주의**: 강한 상승장에서는 RSI 70 이상이 지속될 수 있음. 단독 사용 금지!

### 4.3 MACD

**구성 요소:**
```
MACD선 = EMA(12) - EMA(26)
시그널선 = MACD의 EMA(9)
히스토그램 = MACD선 - 시그널선
```

**매매 신호:**
| 신호 | 조건 | 강도 |
|------|------|------|
| 매수 | MACD선이 시그널선 상향돌파 | 기본 |
| 강한 매수 | 0선 아래에서 골든크로스 | 강력 |
| 매도 | MACD선이 시그널선 하향돌파 | 기본 |
| 강한 매도 | 0선 위에서 데드크로스 | 강력 |

### 4.4 볼린저밴드

**계산식:**
```
중심선 = SMA(20)
상단밴드 = 중심선 + (표준편차 × 2)
하단밴드 = 중심선 - (표준편차 × 2)
```

**활용법:**

| 상황 | 해석 | 전략 |
|------|------|------|
| 밴드 폭 축소 | 변동성 감소, 큰 움직임 예고 | 진입 준비 |
| 상단밴드 이탈 | 과매수 또는 강한 상승 | 추세 확인 후 판단 |
| 하단밴드 이탈 | 과매도 또는 강한 하락 | 반등 신호 대기 |

### 4.5 스토캐스틱

**계산식:**
```
%K = (현재가 - n일 최저가) / (n일 최고가 - n일 최저가) × 100
%D = %K의 m일 이동평균 (보통 n=14, m=3)
```

**매매 신호:**
| 신호 | 조건 |
|------|------|
| 매수 | %K가 20 이하에서 %D 상향돌파 |
| 매도 | %K가 80 이상에서 %D 하향돌파 |

### 4.6 ATR (평균진폭)

**계산식:**
```
TR(진폭) = MAX(고가-저가, |고가-전일종가|, |저가-전일종가|)
ATR = TR의 14일 이동평균
```

**활용:**
- 손절가 설정: 진입가 - (ATR × 2)
- 변동성 측정: ATR이 크면 변동성 높음
- 포지션 사이징: ATR이 크면 작은 포지션

### 4.7 지표 조합 예시

**보수적 조합 (확인형):**
```
매수 조건:
1. RSI < 40 (과매도 근접)
2. MACD 골든크로스
3. 주가 > 20일 이동평균
→ 3개 모두 충족 시 매수
```

**적극적 조합 (선제형):**
```
매수 조건:
1. RSI < 30 (과매도)
2. 볼린저밴드 하단 터치
→ 2개 중 1개 충족 시 매수 (빠른 진입)
```

## 5. 리스크 관리와 손절 전략

### 5.1 왜 리스크 관리가 중요한가?

**손실과 회복의 비대칭:**

| 손실률 | 회복에 필요한 수익률 |
|--------|---------------------|
| -10% | +11.1% |
| -20% | +25.0% |
| -30% | +42.9% |
| -50% | +100.0% |
| -80% | +400.0% |

> **교훈**: 큰 손실 한 번이 작은 이익 여러 번을 무의미하게 만듭니다.

### 5.2 손절(Stop-Loss) 전략

#### 고정 비율 손절

```
손절가 = 진입가 × (1 - 손절률)

예시: 50,000원 진입, 5% 손절
손절가 = 50,000 × 0.95 = 47,500원
```

**손절률 가이드:**

| 투자 스타일 | 손절률 | 특징 |
|-------------|--------|------|
| 보수적 | 3~5% | 작은 손실, 잦은 손절 |
| 표준 | 7~10% | 균형 잡힌 접근 |
| 공격적 | 15~20% | 큰 변동 허용, 적은 손절 |

#### ATR 기반 손절 (추천)

```
손절가 = 진입가 - (ATR × 배수)

예시: 진입가 50,000원, ATR 1,000원, 배수 2
손절가 = 50,000 - (1,000 × 2) = 48,000원
```

**장점**: 변동성에 맞춰 자동 조절

#### 트레일링 스탑

```
상승 시: 손절가도 함께 상승
하락 시: 손절가 유지 (절대 하향 안 함)

예시: 진입 50,000원, 트레일링 5%
- 주가 55,000원 → 손절가 52,250원
- 주가 60,000원 → 손절가 57,000원
- 주가 56,000원 → 손절가 57,000원 유지 → 손절 실행
```

### 5.3 포지션 사이징

#### 고정 비율법

```
1회 투자금 = 총 자본 × 투자비율

예시: 총 자본 1억원, 투자비율 10%
1회 투자금 = 10,000,000원
```

#### 켈리 공식 (Kelly Criterion)

**공식:**
```
f* = (p × b - q) / b

f* = 최적 투자 비율
p = 승률
q = 패률 (= 1 - p)
b = 손익비 (평균이익 / 평균손실)
```

**예시:**
```
승률 55%, 평균이익 10%, 평균손실 7%
b = 10 / 7 = 1.43
f* = (0.55 × 1.43 - 0.45) / 1.43 = 0.236 (23.6%)
```

**실전 적용:**
| 방식 | 비율 | 특징 |
|------|------|------|
| 풀 켈리 | f* × 100% | 이론적 최적, 변동성 매우 큼 |
| 하프 켈리 | f* × 50% | **실전 권장**, 변동성 절반 |
| 쿼터 켈리 | f* × 25% | 보수적, 안정적 |

> **TIP**: 실전에서는 **하프 켈리 이하**를 사용하세요. 풀 켈리는 심리적으로 감당하기 어렵습니다.

### 5.4 최대 낙폭 (MDD) 관리

**MDD 계산:**
```
MDD = (최고점 - 최저점) / 최고점 × 100%

예시: 최고 1억원 → 최저 7천만원
MDD = (1억 - 7천만) / 1억 × 100% = 30%
```

**MDD 한도 설정:**

| 투자 성향 | MDD 한도 | 대응 |
|-----------|----------|------|
| 보수적 | 10% | 한도 도달 시 전량 매도 |
| 표준 | 20% | 한도 도달 시 50% 매도 |
| 공격적 | 30% | 한도 도달 시 리밸런싱 |

### 5.5 일일 손실 한도

```
일일 손실 한도 = 총 자본 × 2%

예시: 총 자본 1억원
일일 손실 한도 = 200만원
→ 하루 200만원 손실 시 당일 매매 중단
```

### 5.6 리스크 관리 체크리스트

```
□ 1회 투자금이 총 자본의 10% 이하인가?
□ 손절가를 미리 설정했는가?
□ 일일 손실 한도를 정했는가?
□ MDD 한도를 정했는가?
□ 연속 손실 시 대응 계획이 있는가?
```

## 6. 매매 주기와 리밸런싱

### 6.1 리밸런싱이란?

포트폴리오의 종목과 비중을 정기적으로 재조정하는 것

**왜 필요한가?**
- 종목 상승/하락으로 비중 불균형 발생
- 새로운 유망 종목 발굴
- 손실 종목 정리

### 6.2 리밸런싱 주기

| 주기 | 장점 | 단점 | 적합한 전략 |
|------|------|------|-------------|
| 일별 | 빠른 대응 | 높은 거래비용 | 단타, HFT |
| 주별 | 트렌드 포착 | 비용 부담 | 모멘텀 |
| **월별** | 균형 | 적당한 비용 | **범용 권장** |
| **분기별** | 낮은 비용 | 느린 대응 | **가치투자 권장** |
| 연별 | 최저 비용 | 매우 느림 | 장기 투자 |

> **결론**: 대부분의 개인 투자자에게는 **월별 또는 분기별** 리밸런싱이 가장 효율적입니다.

### 6.3 리밸런싱 방법

#### 주기형 (Calendar-based)

```
매월 첫 거래일에 리밸런싱 실행

예시:
- 1월 2일: 포트폴리오 재구성
- 2월 1일: 포트폴리오 재구성
- ...
```

#### 밴드형 (Threshold-based)

```
비중이 목표에서 ±5% 이상 이탈 시 리밸런싱

예시: 목표 비중 10%
- 현재 비중 13% → 리밸런싱 실행
- 현재 비중 8% → 대기
```

#### 혼합형 (권장)

```
기본: 분기별 리밸런싱
추가: 비중 ±10% 이탈 시 즉시 리밸런싱
```

### 6.4 거래 비용 고려

```
실제 수익 = 전략 수익 - 거래비용 - 세금 - 슬리피지

거래비용 계산:
- 매수 수수료: 0.015% (증권사별 상이)
- 매도 수수료: 0.015%
- 증권거래세: 0.18% (2025년 기준)
- 슬리피지: 약 0.1~0.3%

1회 왕복 비용 ≈ 0.5%
```

**거래 빈도별 연간 비용:**
| 리밸런싱 주기 | 연간 거래 횟수 | 연간 비용 |
|---------------|----------------|-----------|
| 일별 | 250회 | 125% (손해) |
| 주별 | 52회 | 26% |
| 월별 | 12회 | 6% |
| 분기별 | 4회 | 2% |

## 7. 포트폴리오 구성

### 7.1 종목 수 결정

| 종목 수 | 분산 효과 | 관리 난이도 | 권장 대상 |
|---------|-----------|-------------|-----------|
| 5~10개 | 낮음 | 쉬움 | 집중 투자자 |
| **20~30개** | **적정** | **보통** | **일반 권장** |
| 50개 이상 | 높음 | 어려움 | 기관 투자자 |

> **TIP**: 종목 수가 20개를 넘으면 분산 효과의 증가폭이 크게 감소합니다.

### 7.2 비중 배분 방법

#### 동일 비중 (Equal Weight)

```
각 종목 비중 = 100% / 종목 수

예시: 20종목
각 종목 비중 = 5%
```

**장점**: 단순함, 소형주에 기회 부여
**단점**: 대형 우량주 비중 낮음

#### 시가총액 비중 (Market Cap Weight)

```
종목 비중 = 해당 종목 시총 / 전체 시총 합계

예시: 삼성전자 시총 400조, 포트폴리오 총 시총 1000조
삼성전자 비중 = 40%
```

**장점**: 시장 대표성
**단점**: 대형주 쏠림

#### 역변동성 비중 (Inverse Volatility)

```
종목 비중 ∝ 1 / 변동성

변동성 낮은 종목 → 높은 비중
변동성 높은 종목 → 낮은 비중
```

**장점**: 리스크 균등 배분

### 7.3 섹터 분산

```
권장 섹터 배분:
- 단일 섹터 최대 비중: 30%
- 최소 3개 이상 섹터에 분산
```

**한국 주요 섹터:**
| 섹터 | 대표 종목 | 특징 |
|------|-----------|------|
| IT/반도체 | 삼성전자, SK하이닉스 | 경기 민감 |
| 금융 | KB금융, 삼성생명 | 금리 민감 |
| 바이오 | 삼성바이오, 셀트리온 | 고변동성 |
| 자동차 | 현대차, 기아 | 경기 민감 |
| 화학/소재 | LG화학, 포스코 | 원자재 민감 |

## 8. 전략별 상세 가이드

### 8.1 보수적 전략 (초보자 권장)

**목표**: 연 10~15% 수익, MDD 10% 이내

**전략: 가치 + 퀄리티 복합**

```
종목 선정 기준:
1. PER < 10 (저평가)
2. PBR < 1 (자산가치)
3. ROE > 10% (수익성)
4. 부채비율 < 100% (안정성)
5. 시가총액 상위 200위 이내 (유동성)

포트폴리오:
- 종목 수: 20개
- 비중: 동일 비중 (각 5%)
- 리밸런싱: 분기별
- 손절: 개별 종목 -15%
```

**API 활용:**
```python
# 필요한 API 호출
1. 재무비율 API → PER, PBR, ROE 조회
2. 재무상태표 API → 부채비율 계산
3. 시가총액 순위 API → 유동성 필터링
```

### 8.2 표준 전략 (중급자 권장)

**목표**: 연 15~25% 수익, MDD 20% 이내

**전략: 모멘텀 + 가치 복합**

```
종목 선정 기준:
1. 12개월 수익률 상위 30% (모멘텀)
2. PER 하위 30% (가치)
3. 거래대금 상위 50% (유동성)

포트폴리오:
- 종목 수: 15개
- 비중: 동일 비중
- 리밸런싱: 월별
- 손절: 개별 종목 -10%, 포트폴리오 -15%
```

**매매 규칙:**
```
매수:
- RSI < 50
- MACD 히스토그램 상승 전환
- 20일 이동평균 위

매도:
- RSI > 70
- 12개월 모멘텀 순위 하락 (상위 30% → 50% 이하)
- 손절가 도달
```

### 8.3 공격적 전략 (고급자 전용)

**목표**: 연 30%+ 수익, MDD 30% 감내

**전략: 순수 모멘텀 + 레버리지**

```
종목 선정 기준:
1. 3개월 수익률 상위 10%
2. 신고가 경신 종목
3. 거래량 급증 종목

포트폴리오:
- 종목 수: 10개
- 비중: 모멘텀 강도에 비례
- 리밸런싱: 주별
- 손절: 개별 종목 -7% (빠른 손절)
```

**추가 규칙:**
```
- 시장이 하락 추세(KOSPI < 60일 MA)일 때 현금 50% 유지
- ATR이 평소 2배 이상 시 포지션 축소
- 연속 3회 손절 시 1주 휴식
```

> **경고**: 공격적 전략은 경험이 충분히 쌓인 후 시도하세요.

### 8.4 전략 비교 요약

| 항목 | 보수적 | 표준 | 공격적 |
|------|--------|------|--------|
| 목표 수익 | 10~15% | 15~25% | 30%+ |
| MDD 허용 | 10% | 20% | 30% |
| 종목 수 | 20개 | 15개 | 10개 |
| 리밸런싱 | 분기 | 월 | 주 |
| 손절 | -15% | -10% | -7% |
| 거래 비용 | 낮음 | 보통 | 높음 |
| 시간 투자 | 낮음 | 보통 | 높음 |
| 권장 대상 | 초보자 | 중급자 | 고급자 |

## 9. 백테스팅과 검증

### 9.1 백테스팅이란?

과거 데이터를 사용하여 전략의 성과를 시뮬레이션하는 것

**백테스팅 프로세스:**
```
1. 전략 정의 (규칙 명확화)
2. 과거 데이터 수집
3. 시뮬레이션 실행
4. 성과 분석
5. 전략 수정 (필요시)
6. 실전 적용
```

### 9.2 핵심 성과 지표

| 지표 | 계산 | 좋은 기준 |
|------|------|-----------|
| CAGR | 연평균 복리 수익률 | 15% 이상 |
| MDD | 최대 낙폭 | 20% 이하 |
| 샤프지수 | (수익률-무위험수익률)/변동성 | 1 이상 |
| 승률 | 이긴 거래 / 전체 거래 | 50% 이상 |
| 손익비 | 평균이익 / 평균손실 | 1.5 이상 |

### 9.3 백테스팅 주의사항

**과최적화 (Overfitting) 방지:**
```
나쁜 예: "2020년 3월 23일에 풀매수" → 과거에만 맞는 규칙

좋은 예: "RSI 30 이하일 때 매수" → 일반화 가능한 규칙
```

**생존자 편향 (Survivorship Bias):**
```
문제: 상장폐지된 종목을 제외하고 테스트
결과: 실제보다 좋은 성과 착시

해결: 상장폐지 종목 포함하여 테스트
```

**미래 정보 사용 금지:**
```
문제: 결산 발표일 이전에 실적 데이터 사용
결과: 현실에서 불가능한 전략

해결: 데이터 발표 시점 엄격히 적용
```

### 9.4 검증 기간 설정

```
권장 검증 기간:
- 최소: 5년 (1사이클 이상)
- 권장: 10년 이상
- 이상적: 여러 시장 상황 포함 (상승장, 하락장, 횡보장)
```
