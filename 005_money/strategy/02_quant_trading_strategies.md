# 암호화폐 퀀트 매매 전략 가이드

## 개요

퀀트 트레이딩(Quantitative Trading)은 수학적 모델, 통계적 분석, 알고리즘을 활용하여
거래 결정을 내리는 데이터 기반의 체계적인 거래 방식입니다.

---

## 1. 퀀트 매매의 핵심 특징

### 1.1 정의
- **데이터 기반 의사결정**: 감정이나 직관이 아닌 수치와 패턴에 기반
- **규칙 기반 실행**: 사전에 정의된 수학적 모델과 규칙을 엄격히 준수
- **자동화**: 알고리즘이 매수/매도 시점을 결정하고 실행
- **백테스팅**: 과거 데이터로 전략의 유효성을 검증

### 1.2 암호화폐 시장에서의 퀀트 매매 장점
- **24/7 시장**: 연중무휴 거래 가능 (자동화 필수)
- **높은 변동성**: 빈번한 거래 기회 제공
- **데이터 접근성**: 실시간 가격, 온체인 데이터 등 풍부한 정보
- **낮은 진입장벽**: 개인도 API를 통해 자동매매 가능

---

## 2. 주요 퀀트 매매 전략

### 2.1 추세 추종 전략 (Trend Following)

**개념**: 가격이 한 방향으로 움직이기 시작하면 그 추세가 지속된다는 가정

**주요 기법:**
| 전략 | 설명 | 신호 |
|------|------|------|
| 이동평균 교차 | 단기MA가 장기MA 상향돌파 시 매수 | Golden Cross / Death Cross |
| MACD | MACD선이 시그널선 상향돌파 시 매수 | MACD 히스토그램 |
| DEMA/TEMA | 지연이 적은 지수이동평균 사용 | 추세 방향 전환 |
| 채널 돌파 | 가격이 N일 최고가 돌파 시 매수 | Donchian Channel |

**성과 사례:**
- MACD 기반 비트코인 전략: CAGR 77.24% (Buy & Hold 대비 +16%)
- 전체 시간의 54.44%만 투자하여 변동성 노출 감소

---

### 2.2 평균 회귀 전략 (Mean Reversion)

**개념**: 가격은 장기 평균으로 돌아가려는 성질이 있음

**주요 기법:**
| 전략 | 설명 | 신호 |
|------|------|------|
| 볼린저 밴드 | 가격이 하단밴드 접촉 시 매수, 상단 시 매도 | 밴드 이탈 후 복귀 |
| RSI 과매수/과매도 | RSI 30 이하 매수, 70 이상 매도 | RSI 극값 반전 |
| Z-Score | 가격의 표준편차 이탈 시 역방향 베팅 | Z > 2 또는 Z < -2 |

**적용 조건:**
- 횡보장(Ranging Market)에서 효과적
- 강한 추세장에서는 손실 위험 증가

---

### 2.3 통계적 차익거래 (Statistical Arbitrage)

**개념**: 관련 자산 간의 가격 불균형을 이용한 무위험/저위험 수익

**주요 기법:**
| 전략 | 설명 | 예시 |
|------|------|------|
| 거래소 간 차익거래 | 동일 코인의 거래소 간 가격차 이용 | 업비트 vs 빗썸 김치프리미엄 |
| 페어 트레이딩 | 상관관계 높은 두 자산의 스프레드 거래 | BTC/ETH 스프레드 |
| 삼각 차익거래 | 세 통화 간 환율 불균형 이용 | BTC-ETH-USDT |

**고려사항:**
- 수수료, 슬리피지, 전송 시간이 수익에 직접 영향
- 빠른 실행 속도 필수 (밀리초 단위)

---

### 2.4 변동성 돌파 전략 (Volatility Breakout)

**개념**: 래리 윌리엄스의 전략으로, 전일 변동폭의 일정 비율 이상 돌파 시 진입

**공식:**
```
목표가 = 오늘 시가 + (전일 고가 - 전일 저가) × k
(k = 0.4 ~ 0.6, 보통 0.5 사용)
```

**규칙:**
1. 당일 시가 + 전일 Range × k 돌파 시 매수
2. 다음날 시가에 무조건 청산 (당일 매매)
3. k값과 타겟 코인은 백테스팅으로 최적화

---

### 2.5 모멘텀 전략 (Momentum)

**개념**: 최근 수익률이 높은 자산이 계속 좋은 성과를 보인다는 가정

**주요 기법:**
| 전략 | 설명 |
|------|------|
| 상대 모멘텀 | 여러 코인 중 최근 N일 수익률 상위 코인 매수 |
| 절대 모멘텀 | 자산의 N일 수익률이 양수면 매수, 음수면 현금 보유 |
| 듀얼 모멘텀 | 상대 + 절대 모멘텀 결합 |

---

### 2.6 마켓 메이킹 (Market Making)

**개념**: 호가창에 매수/매도 주문을 동시에 걸어 스프레드 수익

**특징:**
- 유동성 공급자 역할
- 낮은 변동성 시장에서 안정적 수익
- 높은 자본과 고빈도 거래 인프라 필요

---

### 2.7 머신러닝/AI 기반 전략

**개념**: 기계학습 모델을 활용한 가격 예측 및 신호 생성

**활용 모델:**
| 모델 | 용도 |
|------|------|
| LSTM | 시계열 가격 예측 |
| Random Forest | 매수/매도/홀드 분류 |
| XGBoost | 피처 엔지니어링 기반 예측 |
| Reinforcement Learning | 최적 매매 타이밍 학습 |

**주의사항:**
- 과적합(Overfitting) 위험이 매우 높음
- 워크포워드 테스트 필수
- 시장 체제(Regime) 변화에 취약

## 3. 리스크 관리 기법

### 3.1 포지션 사이징

| 방법 | 설명 |
|------|------|
| 고정 비율 | 자본의 일정 % 투자 (예: 2%) |
| ATR 기반 | 변동성 높으면 적게, 낮으면 많이 |
| Kelly Criterion | 최적 베팅 비율 수학적 산출 |

### 3.2 손절매 전략

| 방법 | 설명 |
|------|------|
| 고정 손절 | 진입가 대비 -2% 손절 |
| ATR 손절 | 진입가 - (ATR × 2) |
| 트레일링 스탑 | 최고가 대비 -N% 하락 시 손절 |
| 시간 기반 | N시간/일 내 목표 미달성 시 청산 |

### 3.3 포트폴리오 관리

- **최대 일일 손실 한도**: 일 -3% 초과 시 거래 중단
- **연속 손실 제한**: 3회 연속 손실 시 거래량 축소 또는 중단
- **상관관계 관리**: 유사 자산 과다 노출 방지

## 4. 암호화폐 고유 데이터 활용

### 4.1 온체인 데이터

| 지표 | 설명 | 활용 |
|------|------|------|
| Exchange Netflow | 거래소 입출금량 | 대량 입금 = 매도 압력 |
| Active Addresses | 활성 주소 수 | 네트워크 활동 지표 |
| MVRV Ratio | 시가총액/실현가치 | 고평가/저평가 판단 |
| NVT Ratio | 시가총액/거래량 | P/E와 유사한 밸류에이션 |

### 4.2 선물/파생상품 데이터

| 지표 | 설명 | 활용 |
|------|------|------|
| Funding Rate | 무기한 선물 자금조달율 | 양수 = 롱 과열, 음수 = 숏 과열 |
| Open Interest | 미결제약정 | 급증 시 큰 움직임 예고 |
| Long/Short Ratio | 롱/숏 비율 | 극단적 쏠림 시 반전 신호 |

### 4.3 소셜/센티먼트 데이터

| 소스 | 활용 |
|------|------|
| Twitter/X 언급량 | 관심도 측정 |
| Fear & Greed Index | 시장 심리 지표 |
| Google Trends | 검색 관심도 |

## 5. 퀀트 시스템 구축 요소

### 5.1 필수 구성요소

```
┌─────────────────────────────────────────────────────────┐
│                    퀀트 트레이딩 시스템                    │
```
